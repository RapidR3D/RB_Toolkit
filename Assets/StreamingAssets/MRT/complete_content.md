## CSX Customer Switch Data (CSD): The Process

### Five Steps

**Step One:** Yardmaster or Train Crew receives SPT

**Step Two:** Applies Planned Work per the SPT Order

**Step Three:** Issue Work Accurately & Report Work Timely

**Step Four:** Results = CSD Measure

**Step Five:** Complete the Work Order

---

## CSX CSD Exceptions: Railroad or Customer Reason

### Two Kinds of Exceptions

**Service Exception:**
- Means the cars were not put on the work order, regardless of whether YES or MRT was used to create the work order
- **Full Service Exception** = none of the planned cars were put on the work order
- **Partial Service Exception** = a portion of the planned cars were put on the work order

**Work Order Exception:**
- Indicates car was applied during the work order build and exceptioned by crew using MRT

---

## Customer Switch Data - What Determines the Score

### CSD Measure Looks at Two Factors

**Makes:**
- Place, pull, in-plant, interchange delivery work on the snapshot reported and completed within the defined customer service window

**VS.**

**Misses:**
- Place, pull, in-plant, interchange delivery work on the snapshot that is not completed within the defined customer service window
- Place, pull, in-plant, interchange delivery work on the snapshot that is not completed due to a Railroad Reason exception

## Work Instructions

### Understanding Begin and End Points

Railcars have a Begin Point and an End Point:
- **Begin point** = From Instruction: Where you got the car from, couple up to it, add it to the train
- **End point** = To Instruction: Where you take the car to, uncouple from it, and leave it

Begin & End Point Work Instructions are linked in SPT and on the Work Order.

### Two Letter Abbreviations

To input Add Work on MRT, you must input both:
1. Where you got the car from (origination point of car)
2. Where you expect to leave it (destination point of car)

#### Begin Point - From Instruction
- **PK:** Pickup from Station (CSX yard or Intch)
- **PU:** Pull from Customer

#### End Point - To Instruction
- **PL:** Place at Customer (Spot)
- **SO:** Set Out to Station (CSX yard)
- **DI:** Deliver Interchange to another RR

**Help Desk:** 800-243-7743 Tablets #2, MRT #4  
Call option #4 to report work you are not able to input on MRT

## Terminology

### CSX Station Work at Yards/Terminals
- **PK** - Pickup (add to train to depart with from station)
- **SO** - Set Out (leave at station)
- **LC** - Pickup Equipment of Locomotives & EOTS, shows in separate destination tile of Work Order

### Customer Work at Industry
- **PU** - Pull (remove, take out of)
- **PL** - Place (spot into)
- **IP** - Intraplant Switch (move previously placed cars within the customer facility, stays in customer)

### Interchange Work
- **DI** - Deliver Interchange to other Railroads/Shortlines (take from CSX to designated RR track)
- **PK** - Pickup cars (received interchange from designated RR track to bring online to CSX network)

### Important Notes
Report work accurately with the correct time. It affects car cycles and plant switches, adds/removes cars from customer and yard inventory, and is used to charge the customer.

We are not getting all of the customer place and pull events reported. Note the difference between pickup from station vs pull from customer, and the difference between set out at station vs place at customer.

## Load or Empty Status

### Load Categories
**Loads:**
- **L** - loaded
- **M** - company material loads
- **N** - nobill (loaded car with no billing in CSX systems or unknown L/E)

**Empties:**
- **E** - empty
- **F** - revenue empty

### MRT Input Rules
On MRT, you can only input L or E for the load/empty status. Cars will display the other statuses (M, N, F) to you. Leave them 'as is' unless you need to change the status from either loaded to empty or from empty to loaded.

**Company Directive:** Leave NOBILLS at the Customer until they are billed! Loaded PULLS must be on your work order or come through MRT New Work (don't add work them). It is ok to add work empty pulls from customer. You may need to change the L/E status.

## Getting Started

### Login Procedure

1. Wake up your tablet with the Power Button
2. Tap the screen and swipe up to access unlock screen
3. Input your 4 digit pin to unlock device
4. Select the Mobile Rail Tool Flex app
5. Tap the Login with Okta button
6. Input your RACF ID and password and tap the Sign in button
7. Select the tool you wish to use:
   - Switch Planning Tool (SPT) - Access your work order
   - Mobile Work Order - Complete MRT work order
   - Yard Switching Tool - Move cars within a yard on switchlists: automated or manual
8. Login to Yard Switching Tool to update automated or input manual switchlists
9. Login to SPT, build Outbound Train and submit SPT build
10. Retrieve the work order off the printer
11. Login to Mobile Work Order
12. Depart train
13. Put device in sleep mode and take the tablet with you on the train

3. Select HEAD or REAR; or, to use AFTER, select the After car (behind car) in the To Track and then tap the AFTER button

### Important Note on Yard Direction

### Checking Switch Status
The green check means the switch was done successfully. Tap Refresh List to check for Green Check or Red X.

If you get a Red X instead of a green check, the switch did not go through. You will need to re-input the switch, so that the new switch gets a green check. If you continue to get the Red X, give us a call at the MRT Helpdesk.

**NOTE:** the Red X line will not be removed.

### Reviewing Switched Cars
Tap a row to view switched car details. To return, tap CANCEL.

## SPT: How to Get Your Work Order

### Procedure

1. Request a new Switch Planning Tool (SPT) Report from Crew > 27 > 18, or obtain the original list from printer, counter, or file cabinet
   - OR, you can Request SPT from device if it has not yet generated, or Update SPT if you need a new copy sent to your printer
2. Print any needed track lists from Crew > 27 > 06 > YSTL
3. Log into SPT for your train
4. Select and Add All Cars for train - only add cars you are actually taking on the train
5. Add Locomotives and EOT, if known, and tap Send
6. Adjust the Train Standing Order, and tap Update
7. You may be asked to input Service Exceptions if you cannot take planned cars for placement or interchange
8. You may be asked to input a Printer ID (where the work order will print)
9. A pop-up window informs you that your SPT build has been submitted. Tap OK

### Accessing Switch Planning Tool
1. Tap Switch Planning Tool
2. Enter Train ID
3. **Note:** Only enter an origin milepost if it is a yard job
4. Tap the Create Build icon to begin

### SPT Messages

**Message 1:** "Switch Planning Tool (SPT) has not been generated. Tap Request SPT to generate one now."
- Tap Request SPT to continue. It will create an SPT Report and send a copy to the printer

**Message 2:** "Switch Planning Tool (SPT) has been generated. Tap CONTINUE to use the current SPT. Tap UPDATE SPT to request an updated version for newly arrived inventory or changes in track location."
- Tap Continue OR Tap Update SPT to request a new one or print another copy

### Deleting a Work Order
Once the work order is issued, you may tap the Delete WO icon to delete the work order and start over.

## AEI Reads (Automated Equipment Identification)

### Purpose
Verify your train consist using trackside scanners to ensure work order matches actual train.

### Types of AEI Reads

**Outbound AEI Read**
- When departing origin yard
- Handle BEFORE reporting customer work

**Inbound AEI Read**
- When arriving at destination yard
- Handle BEFORE reporting Set Out

### AEI Read Tabs (Work Left to Right)

**1. AEI Errors**
- Cars with mismatches
- Report or Exception each car

**2. AEI Add Cars**
- Cars scanned but not on work order
- Add Work for each car

**3. All Cars (AEI Standing Order)**
- After resolving errors/adds
- Accept AEI Standing Order

### Resolving AEI Errors

**Option 1: Report the Car**
- Car is actually with you
- AEI scanner missed it
- Verify and report position

**Option 2: Exception the Car**

## Troubleshooting Common Issues

### Cannot Login to MRT
1. Reboot tablet
2. Verify RACF ID and password
3. Check OKTA authentication
4. Call MRT Helpdesk: 800-243-7743 #4

### Work Order Not Found
1. Verify Train ID correct
2. For yard jobs: include milepost
3. Check Incomplete Work Orders
4. Search by WO# instead of Train ID
5. Call MRT Helpdesk if still missing

### Cannot Report Work
1. Check cellular signal
2. Verify MQTT Connected: Yes
3. Try backing out and re-entering
4. Restart MRT app
5. Call MRT Helpdesk

### Logout Not Working
1. Verify 100% completion
2. Check MQTT status
3. Check for error messages
4. Force close and reopen
5. Call MRT Helpdesk - DO NOT just walk away

### Work Not Posting to System
1. Did you actually tap Logout?
2. Did you see "Login with Okta" confirmation?
3. Check MQTT was Yes/Yes
4. Verify cellular signal present
5. Call MRT Helpdesk to verify
